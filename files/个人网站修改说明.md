# 个人网站（Academic Pages / Jekyll）修改说明

本仓库基于 [Academic Pages](https://github.com/academicpages/academicpages.github.io) 模板，通过 **GitHub Pages** 发布。站点地址：<https://haodong-lei-ray.github.io/>

---

## 1. 改首页与自我介绍

- 文件：`_pages/about.md`
- 该页 `permalink: /`，即访客打开站点根路径时看到的内容。
- 修改后直接保存，推送到 GitHub 后由 Pages 重新构建（通常 1～3 分钟）。

可改内容示例：教育背景、论文列表、研究兴趣、联系方式、外链（如知乎）等。

---

## 2. 博客（当前已关闭）

本站已去掉站内 `_posts` 博文与顶部「Blog Posts」入口。若以后需要恢复：

- 在 `_posts/` 中按 `YYYY-MM-DD-slug.md` 添加文章（需 YAML 头信息）；
- 将 `_data/navigation.yml` 里 Blog Posts 一项取消注释；
- 可从 Academic Pages 模板中拷回 `_pages/year-archive.html` 作为按年归档页。

---

## 3. 站点全局配置（作者、标题、社交链接）

- 文件：`_config.yml`
- 常用项：
  - `title`、`description`、`url`
  - `author:` 下的 `name`、`bio`、`location`、`employer`、`email`、`github`、`googlescholar` 等
- **注意**：改 `_config.yml` 后，本地预览需**重启** `jekyll serve`。

---

## 4. 上传 PDF 等静态文件

- 目录：`files/`
- 访问地址：`https://haodong-lei-ray.github.io/files/你的文件名.pdf`

**首页简历按钮**：`files/chinese_cv.pdf`（中文）、`files/english_cv.pdf`（英文），对应线上地址示例：`https://haodong-lei-ray.github.io/files/english_cv.pdf`（与 GitHub 仓库 `files/` 下文件名一致；`github.com/.../blob/...` 是网页预览，下载用 Pages 域名）。源文件放在个人资料根目录：`chinese_cv.pdf`、`english_cv.pdf`。更新后运行 `python scripts/write_chinese_cv_pdf.py`，或手动复制到 `files/`，再 `git add` / `push`。
- 在 Markdown 里引用：`[下载简历]({{ site.baseurl }}/files/resume.pdf)`  
  （若 `baseurl` 为空，也可写 `/files/resume.pdf`。）

---

## 5. 论文、报告、课程（可选）

模板还支持集合目录（需在 `_config.yml` 的 `collections` 中已启用）：

| 内容     | 目录            |
|----------|-----------------|
| 论文     | `_publications/` |
| 演讲     | `_talks/`        |
| 教学     | `_teaching/`     |
| 作品集   | `_portfolio/`    |

每篇一般为独立 `.md`，具体字段可参考目录内已有范例。

---

## 6. 本地预览（可选）

1. 安装 Ruby、Bundler（Windows 可用 WSL 或参考模板 README）。
2. 在仓库根目录执行：`bundle install`
3. 执行：`bundle exec jekyll serve -l -H localhost`
4. 浏览器打开：<http://localhost:4000>

或使用仓库内 **Docker**：`docker compose up`，同样访问 4000 端口。

---

## 7. 发布流程（必做）

```bash
git add .
git commit -m "描述你的修改"
git push origin main
```

默认分支构建完成后，线上站点即更新。若 Pages 报错，在仓库 **Settings → Pages** 查看构建日志。

---

如有模板升级需求，可阅读仓库根目录 `README.md` 中的官方说明与许可信息。
